## 🎯 一句话说清

**职责分离 + 小任务切片 + 隔离执行 = 多 Agent 协作的智能工程体系**

---

## ✨ 核心特性

| 特性 | 描述 |
|-----|------|
| 🎯 | **职责分离** - 老明规划，小明干活，各司其职 |
| 📋 | **任务文档即契约** - 每个任务都有明确的"菜谱" |
| 🔢 | **3-4 文件切片** - 每次只改 3-4 个文件，易评审 |
| 🏠 | **Git Worktree 隔离** - 每个任务有独立工作间，互不干扰 |
| 👥 | **固定团队复用** - 匠人们持续工作，效率越来越高 |
| 🔄 | **按需任务创建** - 做好一个再分配下一个，不浪费 |
| ⭐ | **强制测试驱动** - 每个小任务必须包含测试 |
| ✅ | **验收标准明确** - 任务不完成不算完 |

## 🚀 快速开始

### 第一步：阅读视觉化教学

> 不想看长文档？直接学建筑比喻！

```bash
# 查看完整的施工故事
cat docs/harness-pro-visual-teaching.md
```

### 第二步：理解核心概念

| 角色 | 比喻 | 职责 |
|-----|------|------|
| **Main Agent** | 总建筑师老明 | 画图纸、分任务、查质量 |
| **Worker Agent** | 匠人（小明/小红/小刚） | 按任务书完成工作 |
| **Task Document** | 菜谱/任务书 | 明确做什么、不做什么 |
| **Git Worktree** | 独立工作间 | 互不干扰的工作环境 |
| **main branch** | 主楼 | 最终成果所在 |

### 第三步：开始使用

```bash
# 初始化项目
cd your-project

# 创建目录结构
mkdir -p .claude/skills/your-main
mkdir -p .claude/skills/your-worker
mkdir -p .worktrees
mkdir -p docs/tasks
```

## 🌟 为什么选择 Harness-Pro

### 传统开发的问题

```
问题 1：大分支，几千行代码，评审痛苦
       ↓
问题 2：多人同时改同一目录，容易冲突
       ↓
问题 3：代码写完了才发现测试不足
       ↓
问题 4：项目进度不透明
       ↓
问题 5：代码回滚困难
```

### Harness-Pro 的解决方案

```
解决 1：小任务切片（3-4 文件）
       ↓
解决 2：独立工作区，互不干扰
       ↓
解决 3：每个任务强制测试
       ↓
解决 4：任务状态清晰可见
       ↓
解决 5：小任务独立回滚
```
