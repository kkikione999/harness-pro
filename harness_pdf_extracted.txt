1.2 核心理念
Humans steer. Agents execute.
（人类掌舵，Agent 执行）
关键转变：
• 从” 编写代码” → “设计环境、明确意图、构建反馈循环”
• 从” 工程师打字” → “工程师编排”
• 从” 人类审查每个 PR” → “Agent-to-Agent 审查”

┌─────────────────────────────────────────────────────────────┐
│ Task Assignment Hierarchy │
├─────────────────────────────────────────────────────────────┤
│ │
│ Level 1: Thread Level (会话级) │
│ ┌─────────────────────────────────────────────────────┐ │
│ │ Thread: 用户和 agent 之间持续会话的持久容器 │ │
│ │ • 创建、恢复、分叉、归档 │ │
│ │ • 持久化事件历史，支持客户端重连 │ │

│ ↓ │

│ ┌─────────────────────────────────────────────────────┐ │

│ │ • 包含表示中间步骤的 item 序列 │ │
│ │ • 在 agent 完成输出时结束 │ │
│ └─────────────────────────────────────────────────────┘ │
│ ↓ │
│ Level 3: Item Level (原子操作级) │
│ ┌─────────────────────────────────────────────────────┐ │
│ │ Item: 输入/输出的原子单元，有明确生命周期 │ │
│ │ • item/started → item/*/delta → item/completed │ │
│ └─────────────────────────────────────────────────────┘ │
