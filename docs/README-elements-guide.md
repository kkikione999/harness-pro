## 🎯 七大核心元素解析

### 1️⃣ 标题 + Badge 行

#### 设计意图

```
目的：快速传达项目状态和质量
位置：文件第一行
元素：
  - 项目名称 + Emoji 图标
  - 一排 Badge（许可、Stars、Forks、Issues、PRs）
```

#### 实现代码

```markdown
# 🏗️ Harness-Pro

[![License: MIT](https://img.shields.io/badge/License-MIT-yellow.svg?style=flat-square)](LICENSE)
[![GitHub stars](https://img.shields.io/github/stars/username/repo?style=flat-square)](../../stargazers/username/repo)
[![GitHub forks](https://img.shields.io/github/forks/username/repo?style=flat-square)](../../network/members/username/repo)
```

#### Badge 配置说明

| 参数 | 值 | 说明 |
|-----|-----|------|
| `style` | `flat-square` | 方形扁平风格 |
| `style` | `for-the-badge` | 适合 Badge 的大圆角 |
| `color` | `blue/green/yellow` | 自定义颜色 |

### 2️⃣ 品牌宣言（引用框）

#### 设计意图

```
目的：一句话说清项目核心价值
位置：标题下方
风格：建筑比喻 + 口语化表达
```

#### 实现代码

```markdown
> **建筑师的施工蓝图：多人协作的智能工程管理体系**
> 让复杂工程任务变成像盖房子一样有条不紊
```

#### 技巧要点

- 使用 `>` 创建引用框
- 粗体 `__text__` 突出关键词
- 两行：标题 + 副标题
- 语言简洁有力

### 3️⃣ 角色 ASCII 图形

#### 设计意图

```
目的：视觉化抽象概念
位置：简介部分
风格：框架式建筑结构
```

#### 实现代码

```markdown
┌─────────────────────────────────────────┐
│  🏛️ 总指挥中心                        │
│          Main Agent = "总建筑师老明"    │
└─────────────────────────────────────────┘
```

#### 构建技巧

- 使用 `┌` `│` `─` `└` 绘制边框
- 保持宽度一致
- 用 Emoji 作为图标点缀
- 层级缩进表示关系

---

### 4️⃣ 快速开始（三步指引）

#### 设计意图

```
目的：降低上手门槛
位置：角色图下方
结构：三步递进
```

#### 实现代码

```markdown
## 🚀 快速开始

### 第一步：阅读教学指南
> 不想看长文档？直接学建筑比喻！

### 第二步：理解核心概念
| 角色 | 比喻 | 职责 |
| --- | --- | --- |
| Main Agent | 总建筑师老明 | 画图纸、分任务、查质量 |

### 第三步：开始使用
```bash
# 初始化项目
cd your-project
mkdir -p .claude/skills
```
```

#### 设计原则

- 使用 Emoji 标记步骤（1️⃣ 2️⃣ 3️⃣）
- 每步都有清晰目标
- 包含可复制代码
- 进度递进：理解→学习→使用

### 5️⃣ 核心亮点（表格）

#### 设计意图

```
目的：快速了解项目特点
位置：快速开始下方
结构：两列表格 + Emoji
```

#### 实现代码

```markdown
## ✨ 核心亮点

| 特性 | 描述 |
| ----- | ----- |
| 🎯 **职责分离** | 老明规划，小明干活，各司其职 |
| 📋 **任务文档即契约** | 每个任务都有明确的"菜谱" |
| 🔢 **3-4 文件切片** | 每次只改 3-4 个文件，易评审 |
```

#### 技巧要点

- 首列留空给 Emoji
- 每行一个独特图标
- 标题用粗体 `__标题__`
- 描述两句话说明
- 按重要性排序

### 6️⃣ 对比展示（问题→解决）

#### 设计意图

```
目的：突出差异化价值
位置：核心亮点下方
结构：并列对比
```

#### 实现代码

```markdown
## 🎓 为什么选择 Harness-Pro

### 传统开发问题

```
问题：
├─ 大分支，几千行代码，评审痛苦
├─ 多人同时改同一目录，容易冲突
└─ 代码写完了才发现测试不足
```

### Harness-Pro 的解决方案

```
解决：
├─ 小任务切片（3-4 文件）
├─ 独立工作区，互不干扰
└─ 每个任务强制测试
```
```

#### 设计原则

- 明确标注"问题"和"解决"
- 使用树形结构 `├─` `└─`
- 并列对比，一目了然
- 语义化命名

### 7️⃣ 文档导航（分层链接）

#### 设计意图

```
目的：引导用户深入学习
位置：对比展示下方
结构：分层分类
```

#### 实现代码

```markdown
## 📚 文档导航

### 入门阶段
| 文档 | 描述 | 像什么 |
| ----- | ----- | ------ |
| **[视觉化教学](...)** | 用建筑比喻讲解 | 一部完整的施工故事 |
| **[入门指南](...)** | 从零开始学 | 三步上手指南 |

### 深入阶段
| 文档 | 描述 | 像什么 |
| ----- | ----- | ------ |
| **[深度分析](...)** | 六个专家视角 | 架构师、工程师、程序员等 |
```

#### 分类原则

- 按学习阶段分类
- 每个链接有描述
- 说明"像什么"帮助判断
- 使用粗体标记链接

## 🎨 综合设计原则

### 原则 1：视觉一致性

```
Emoji 使用：
- 功能类：🎯 ✨ 🚀
- 文档类：📚 📖 📋
- 导航类：↗️ ↘️ ↖️
- 状态类：✅ ⏳ 🚧
```

### 原则 2：层次清晰

```
层级结构：
# H1: 项目名称
## H2: 主要章节
### H3: 子章节
#### H4: 细节说明
```

### 原则 3：节奏感

```
阅读节奏：
1. 标题（抓眼球）
2. 简述（激发兴趣）
3. 图形（视觉理解）
4. 详情（深入了解）
5. 行动（开始使用）
```

### 原则 4：记忆点

```
记忆设计：
- 口诀化表达
- 对比展示
- 比喻贯穿
- 可视化图表
```

## 🛠️ Markdown 速查表

| 效果 | 语法 | 说明 |
|-----|------|------|
| 粗体 | `__text__` | 双下划线 |
| 斜体 | `_text_` | 单下划线 |
| 删除线 | `~~text~~` | 波浪线 |
| 代码 | ``code``` | 单反引号 |
| 代码块 | ` ``` ` ... ` ``` ` | 三反引号 |
| 链接 | `[text](url)` | 方括号+圆括号 |
| 图片 | `
![alt](url)` | 感叹号 |
| 引用 | `> text` | 大于号 |
| 分隔线 | `---` | 三个短横 |
| 标题 | `# H1` ~ `###### H6` | 1-6个# |
| 列表 | `- item` | 短横线 |
| 表格 | ` \| ` | 竖线 |
| Emoji | `:emoji:` | 冒号+名称 |
