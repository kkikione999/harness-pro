## 🎯 设计亮点分析

### 亮点 1：建筑比喻贯穿全文

```
问题：抽象概念难理解
解决：用建筑工地场景比喻

┌─────────────────────────────────┐
│  老明（总建筑师）              │
│  = Main Agent                  │
│  职责：画图纸、分任务、查质量   │
└─────────────────────────────────┘

┌─────────────────────────────────┐
│  小明（石匠）                  │
│  = Worker Agent               │
│  职责：按任务书完成工作         │
└─────────────────────────────────┘
```

### 亮点 2：视觉化表格

```markdown
| 角色 | 比喻 | 职责 |
|------|------|------|
| Main Agent | 总建筑师老明 | 画图纸、分任务、查质量 |
| Worker Agent | 匠人（小明/小红/小刚） | 按任务书完成工作 |
```

### 亮点 3：对比展示

```markdown
## 传统开发问题

问题：
├─ 大分支，几千行代码，评审痛苦
├─ 多人同时改同一目录，容易冲突
└─ 代码写完了才发现测试不足

## Harness-Pro 的解决方案

解决：
├─ 小任务切片（3-4 文件）
├─ 独立工作区，互不干扰
└─ 每任务强制测试
```

### 亮点 4：口诀记忆

```markdown
核心概念记忆口诀：

1. 老明规划，小明干活 → 职责分离
2. 任务书是菜谱，按着做 → 遵循契约
3. 三四文件，大小正好 → 任务切片
4. 工作区独立，互不打扰 → 隔离执行
5. 固定团队，复用效率 → 团队管理
```

## 📝 使用的 Markdown 技巧清单

| 技巧 | 示例 | 用途 |
|------|------|------|
| **Emoji 作为标题** | `## 🚀 快速开始` | 视觉引导 |
| **引用框** | `> __品牌宣言__` | 突出重点 |
| **表格** | `| 🎯 | 核心能力 |` | 结构化信息 |
| **代码块** | ` ```bash ... ``` ` | 命令展示 |
| **分隔线** | `---` | 章节分割 |
| **粗体** | `__文本__` | 强调内容 |
| **斜体** | `_文本_` | 说明文字 |
| **ASCII 图形** | 树形结构图 | 可视化展示 |
| **列表** | `- 项目` | 清单展示 |
| **链接** | `[文本](url)` | 导航跳转 |

## 🎭 使用的比喻总结

| 概念 | 比喻 | 画面感 |
|------|------|--------|
| **Main Agent** | 总建筑师老明 | 戴眼镜、穿西装、手握图纸 |
| **Worker Agent** | 匠人们（小明/小红/小刚） | 各有工装、各拿工具 |
| **Task Document** | 菜谱/任务书 | 贴在墙上、步骤清晰 |
| **Git Worktree** | 独立工作间 | 每人有自己的工作空间 |
| **main branch** | 主楼 | 最终成果的展示地 |
| **任务拆分** | 盖房子分步 | 地基→墙→屋顶→装修 |
| **固定团队** | 常驻员工 | 不招临时工、效率高 |
