## BUG IDENTIFIED

File: /tmp/test-api-v1-baseline/src/main.js
Function: handleUsers()
Problem: Always returned HTTP 500 with fake "Database unavailable" error,
        despite hardcoded users data being available.

Root cause: Intentional broken stub — hardcoded DB_CONFIG.host = "wrong-host.local"
            caused simulate_db_failure() behavior in handleUsers().

## FIX APPLIED

Changed handleUsers() to return the hardcoded users array directly:

BEFORE:
  res.writeHead(500, { 'Content-Type': 'application/json' });
  res.end(JSON.stringify({ error: 'Database unavailable', ... }));

AFTER:
  res.writeHead(200, { 'Content-Type': 'application/json' });
  res.end(JSON.stringify({ users: users }));

## SCAFFOLDING BUGS FIXED DURING EVALUATION

The v1 scaffolding templates had path resolution bugs that were fixed:

1. Double .harness path: Templates used $PROJECT_ROOT/.harness/... but
   PROJECT_ROOT already resolved to .harness/ (scripts are 1 level deep).
   Fix: Changed to $PROJECT_ROOT/... (PID file, LOG_DIR paths).

2. BASH_SOURCE resolution: When run as ./start.sh from controllability/,
   dirname(BASH_SOURCE[0]) = "." so PROJECT_ROOT was wrong.
   Fix: Changed to SCRIPT_DIR="$(cd "$(dirname "${BASH_SOURCE[0]}")" && pwd)"
        then PROJECT_ROOT="$(cd "$SCRIPT_DIR/../.." && pwd)".

3. Default port mismatch: verify.sh/health.sh defaulted to :8080 but API
   server runs on port 3000. Fix: Updated default to :3000.

4. Missing project pattern: start.sh only checked package.json/go.mod/Makefile
   but this project has src/main.js. Fix: Added src/main.js detection.

5. stop.sh pkill pattern: Was pkill -f "test_api_v1_baseline" which matches
   .harness directory. Fix: Changed to pkill -f "node.*main.js".

6. Makefile cd path: "cd .. && ./controllability/start.sh" was wrong when
   run from controllability/. Fix: Changed to ./start.sh (same dir).

7. Makefile exit code capture: "[ $$EXIT_CODE -eq 0 ]" failed when EXIT_CODE
   was empty due to Makefile multi-line issues. Fix: Combined into single
   shell command with semicolons.

8. test-auto.sh path: Same BASH_SOURCE issue as above. Fix: Same approach.
